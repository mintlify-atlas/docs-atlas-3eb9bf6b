---
title: "Avatar"
description: "Image and text-based avatar components with six sizes and four status indicator states."
---

TailAdmin ships two avatar components: `Avatar` for image-based avatars and `AvatarText` for initial-based fallback avatars.

```tsx
import Avatar from "@/components/ui/avatar/Avatar";
import AvatarText from "@/components/ui/avatar/AvatarText";
```

## Avatar (image)

### Props

<ParamField path="src" type="string" required>
  The URL of the avatar image passed to Next.js `<Image>`.
</ParamField>

<ParamField path="alt" type="string" default="User Avatar">
  Alternative text for the image.
</ParamField>

<ParamField path="size" type="xsmall | small | medium | large | xlarge | xxlarge" default="medium">
  Controls the diameter of the avatar circle.

  | Value | Diameter |
  |---|---|
  | `xsmall` | 24 px |
  | `small` | 32 px |
  | `medium` | 40 px |
  | `large` | 48 px |
  | `xlarge` | 56 px |
  | `xxlarge` | 64 px |
</ParamField>

<ParamField path="status" type="online | offline | busy | none" default="none">
  Renders a colored dot indicator in the bottom-right corner of the avatar.

  | Value | Color |
  |---|---|
  | `online` | `bg-success-500` (green) |
  | `offline` | `bg-error-400` (red) |
  | `busy` | `bg-warning-500` (amber) |
  | `none` | No indicator rendered |
</ParamField>

### Usage examples

<Tabs>
  <Tab title="Sizes">

## AvatarText (initials fallback)

Use `AvatarText` when you have a user's name but no image. It generates up to two initials and assigns a consistent pastel color derived from the name string.

### Props

<ParamField path="name" type="string" required>
  The full name of the user. Initials are derived by taking the first character of each word (up to two characters).
</ParamField>

<ParamField path="className" type="string">
  Additional Tailwind classes merged onto the avatar container. The component has a fixed `h-10 w-10` base size by default.
</ParamField>

### Usage example

```tsx
<AvatarText name="John Doe" />
<AvatarText name="Alice" />
<AvatarText name="Bob Smith" className="h-12 w-12" />
```

The color is derived deterministically from the character codes in `name`, so the same name always produces the same color. Available palette entries include brand, pink, cyan, orange, green, purple, yellow, and error tones.

<Tip>
  Combine `Avatar` and `AvatarText` in a conditional: render `Avatar` when `src` is available and fall back to `AvatarText` when it is not.
</Tip>
