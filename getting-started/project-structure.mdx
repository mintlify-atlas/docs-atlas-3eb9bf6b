## Folder reference

### `public/`

Static assets served directly by Next.js. The `images/` subfolder holds all image files referenced by the template (logos, avatars, chart placeholders, etc.).

### `src/app/`

The Next.js App Router root. The root `layout.tsx` wraps the entire application with the `ThemeProvider` and `SidebarProvider` context providers and loads the `Outfit` Google Font. `globals.css` holds all Tailwind CSS v4 theme tokens and global styles.

### Route groups

TailAdmin uses [Next.js route groups](https://nextjs.org/docs/app/building-your-application/routing/route-groups) (folders wrapped in parentheses) to share layouts without affecting the URL structure.

#### `(admin)`

All standard dashboard pages. The `layout.tsx` inside this group renders the collapsible **AppSidebar**, the **AppHeader**, and the **Backdrop** overlay. Pages in `(others-pages)` and `(ui-elements)` inherit this layout automatically.

| Sub-group | What it contains |
|---|---|
| `(others-pages)` | Charts, forms, tables, calendar, user profile, and a blank page template |
| `(ui-elements)` | Isolated UI component demo pages вЂ” alerts, avatars, badges, buttons, images, modals, and videos |

#### `(full-width-pages)`

Pages that need a different shell вЂ” no sidebar or header. The group has its own `layout.tsx`.

| Sub-group | What it contains |
|---|---|
| `(auth)` | Sign-in and sign-up pages with their own centered layout |
| `(error-pages)` | The custom 404 error page |

### `src/components/`

All reusable React components, organised by domain:

| Folder | Contents |
|---|---|
| `auth/` | Sign-in and sign-up form components |
| `calendar/` | Full-calendar integration component |
| `charts/` | ApexCharts wrappers (line, bar, pie, etc.) |
| `common/` | Shared primitives such as breadcrumbs and loaders |
| `ecommerce/` | Dashboard widgets and stat cards |
| `form/` | Input, select, checkbox, radio, date-picker, and other form elements |
| `header/` | Header-specific components (user menu, notifications) |
| `tables/` | Table components with sorting and pagination |
| `ui/` | General UI elements вЂ” alerts, avatars, badges, buttons, modals, etc. |
| `user-profile/` | Profile page sections |
| `videos/` | Video embed components |

### `src/context/`

React context providers:

- **`ThemeContext.tsx`** вЂ” manages the `light` / `dark` theme state and persists the preference to `localStorage`.
- **`SidebarContext.tsx`** вЂ” manages expanded/collapsed sidebar state and mobile-open state.

Both providers are mounted at the root layout level so every page in the app can access them.

### `src/hooks/`

Custom React hooks:

- **`useGoBack.ts`** вЂ” programmatic back-navigation helper.
- **`useModal.ts`** вЂ” open/close toggle state for modal dialogs.

### `src/icons/`

SVG icon files imported as React components via `@svgr/webpack`. The `index.tsx` file re-exports every icon so you can import from a single location:

```typescript
import { BellIcon, EnvelopeIcon } from "@/icons";
```

### `src/layout/`

Top-level shell components rendered by the `(admin)` group layout:

- **`AppHeader.tsx`** вЂ” the sticky top navigation bar.
- **`AppSidebar.tsx`** вЂ” the collapsible side navigation.
- **`Backdrop.tsx`** вЂ” the semi-transparent overlay shown behind the sidebar on mobile.
- **`SidebarWidget.tsx`** вЂ” an optional promotional or info widget rendered at the bottom of the sidebar.

### Root config files

| File | Purpose |
|---|---|
| `next.config.ts` | Next.js configuration; adds `@svgr/webpack` for SVG imports |
| `postcss.config.js` | PostCSS config that loads the `@tailwindcss/postcss` plugin |
| `tsconfig.json` | TypeScript config with the `@/*` path alias pointing to `src/` |
| `package.json` | Dependencies and npm scripts |
